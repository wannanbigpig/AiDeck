# AiDeck 更新记录

本文档用于永久记录每次版本更新内容。消息中心的 `announcements.json` 只保留当前需要提示的最新版本更新公告；发布下一个版本时，应删除或替换上一版本的更新公告，但不要删除本文档中的历史记录。

## v1.0.9

- Codex 订阅到期时间改为优先读取具体 workspace 的 `entitlement.expires_at`，并在 Node 请求遇到 Cloudflare challenge 时通过原生 `fetch` 重试；远程资料缺失时仍保留 JWT claim 兜底。
- 修复同邮箱个人 `Plus` 与团队 `Team` workspace 订阅时间串用，以及重复账号 ID 期间多张卡片同时显示“当前”的问题。
- Codex 设置新增默认关闭的“5 小时配额控制”；关闭时仍展示 5 小时配额，但不参与自动切号和系统级预警，每周配额逻辑保持不变。
- 补充 workspace 订阅解析、远程重试、唯一当前账号和 5 小时配额控制的回归测试。
- 同步更新项目版本号、README 版本徽章和消息中心公告内容。

## v1.0.8

- 修复 Codex 同一 `account_id` 下个人 `Plus` workspace 与团队 `Team` workspace 可能被错误归并的问题；账号资料、套餐和 workspace 名称现在会优先按 `account_id + organization_id` 精确匹配。
- 修复 Codex `accounts/check` 资料解析和套餐提取链路，刷新配额、同步账号资料时不再把另一 workspace 的 `plan_type`、角色或 workspace 名称串到当前账号。
- 补充同 `account_id`、不同 `organization_id` 场景的回归测试，覆盖套餐解析、workspace 资料解析和 profile 拉取链路。
- 同步更新项目版本号、README 版本徽章和消息中心公告内容。

## v1.0.7

- 修复 uTools 生产构建与导入链路：`dist/plugin.json` 不再保留开发入口，同时为历史页面 chunk 增加 fallback，降低 `file://` 插件因旧缓存入口命中缺失 chunk 导致的白屏问题。
- Codex 账号卡片新增“重置使用次数”能力，可读取剩余重置次数、展示最近到期时间，并在确认后执行一次重置。
- 修复 Codex 重置次数相关接口刷新 token 后未持久化的问题，避免后续刷新重置次数或消耗重置次数时继续使用旧 token。
- 同步账号信息按钮改为真实执行“同步账号资料 + 刷新配额”，并补齐重置次数与账号资料字段的刷新。
- 修正重置次数最近到期时间的展示逻辑，不再把多次重置中的最远到期时间误当作最近到期时间显示。
- 调整 Codex JSON 导入说明与能力口径，不再支持 ChatGPT Web 登录 session JSON；请改用 OAuth、API Key 或 `auth.json` 导入。
- 优化 Codex 空账号状态下的会话管理入口，即使尚未添加账号，也可以直接进入会话管理页。
- 同步更新版本号、README 版本徽章与消息中心公告内容。

## v1.0.6

- Codex Token / JSON 导入说明已调整，明确支持当前应用导出的 JSON、按示例手动拼接的 JSON，以及 `chatgpt.com/api/auth/session` 返回的 JSON。
- 修复通过 `chatgpt.com/api/auth/session` 导入的 Codex 账号被误判为“当前本地账号未导入”的问题，同一账号会根据 `user_id`、`account_id`、邮箱和 token 信息综合识别。
- 优化 Codex 自动切号后的客户端重启判断，macOS 下可识别 Codex App 主进程、Helper 和 app-server，减少切号成功但未自动重启的情况。
- Codex 会话管理新增压缩包导入导出，支持导出选中会话；未选择时默认导出当前可见会话。
- 导出改为后台任务，提供进度、导出历史、本机定位历史包、删除历史包和终止运行中导出的能力。
- 导入支持选择默认 `~/.codex` 或账号绑定实例，重复导入同源会话时保留目标中更新的版本。
- 修复导入默认 `~/.codex` 时可能生成自引用 `rules` / `skills` 软链接的问题，避免 Codex App 打开导入会话时报错。
- Codex CLI 账号独立实例启动时同步 `config.toml`，让 MCP 配置随实例可用；`auth.json` 仍保持账号实例独立，避免凭证串用。
- 优化会话列表工具栏、目录全选 / 半选样式、导入导出图标和导出历史状态展示。
- 同步更新公告内容。

## v1.0.4

- Codex 设置新增 CLI 命令位置配置，支持自动检测、手动选择和自定义输入，后续账号启动、继续会话和唤醒任务统一使用该命令。
- Codex CLI 设置中显示检测到的完整版本输出，便于确认当前使用的 `codex-cli` 版本。
- 修复从 AiDeck 启动 Codex CLI 时，绝对 `codex` 路径仍可能被 `PATH` 中旧 Node 执行的问题，并精简终端启动命令中的环境变量。
- 立即唤醒改为后台异步执行，点击后立即显示“唤醒中”状态，并记录最近结果、耗时、上次执行、下次触发和触发方式。
- Codex 唤醒使用独立 `CODEX_HOME` 和独立会话来源，避免唤醒任务读取普通账号实例规则或污染普通会话。
- 优化 Codex 会话管理：默认实例历史会话与 Codex 桌面项目列表口径对齐，唤醒会话统一聚合展示，顶部工具栏控件高度对齐。
- 同步更新项目版本号、README 版本徽章和公告内容。

## v1.0.3

- 新增后台任务运行提醒，说明定时唤醒、自动刷新等任务依赖 uTools 插件保持后台运行。
- 后台任务提醒改为放入消息通知中心，不再占用首页仪表盘空间。
- 消息通知中心的全部已读、刷新、关闭操作移到标题栏右侧，并统一为紧凑图标按钮。
- 刷新按钮增加加载旋转反馈，点击后至少展示 1 秒加载状态。
- Codex CLI 账号绑定实例不再共享会话数据，仅共享 `skills`、`rules`、`vendor_imports/skills` 和 `AGENTS.md`，避免多实例会话索引与 `state_5.sqlite` 状态不一致。
- 新增 Codex 会话管理入口，可在账号页切换到会话视图，按工作区查看默认实例和账号绑定实例中的本地会话。
- 支持预览 Codex 会话消息，Markdown 内容、代码块、表格和图片附件可直接查看，内部指令和工具日志默认折叠到调试信息中。
- 支持继续会话、归档 / 取消归档、移入回收站、从回收站恢复或彻底删除会话。
- 支持将会话复制或移动到指定账号绑定实例，并在复制时生成新的会话 ID，避免多账号实例复用同一会话索引。
- 新增会话索引清理能力，可移除缺失会话文件残留的 SQLite / `session_index.jsonl` 索引记录。
- 使用说明从页面内折叠卡片改为标题旁弹窗入口，侧边栏默认收起，设置页返回会回到进入设置前的平台。
- 同步更新项目版本号、README 版本徽章和公告内容。

## v1.0.2

- 项目收敛为 uTools 单端，移除 Electron Desktop、旧根入口和历史 preload 转发层。
- 新增消息通知中心，支持远程公告、未读角标、置顶、刷新、全部已读和自动弹窗。
- Codex / Gemini 账号卡片新增 CLI 启动按钮，并支持默认终端设置。
- Codex CLI 启动改为账号绑定独立实例，首次点击创建独立 `CODEX_HOME`，后续复用；删除账号时同步删除绑定实例。
- Codex CLI 多账号实例共享静态配置目录，便于复用规则和 skills。
- Codex 账号卡片新增唤醒任务按钮，支持按账号设置启用 / 停用、每日 / 每周 / 间隔 / 配额重置 / 启动后调度、预设或自定义模型，并可立即用账号绑定实例执行 `codex exec` 轻量唤醒；模型预设支持 `GPT-5.5`、`GPT-5.4`、`GPT-5.4-Mini`、`GPT-5.3-Codex`、`GPT-5.2`，Pro 账号额外显示 `GPT-5.3-Codex-Spark`。
- Codex 配额展示补齐 5 小时、每周、剩余额度和额外模型额度。
- Codex 卡片背面新增订阅到期展示。
- 配额自动刷新间隔支持 1-60 分钟细粒度设置。
- 新增系统级配额预警通知。
- 优化请求日志写入，降低高频日志场景下的同步 IO 压力。
- 修复操作日志弹窗懒加载边界、批量刷新返回值、导入按钮文案等问题。
